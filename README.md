#カレンダーアプリ

環境構築から日々の開発の流れまで、初心者でもわかるように手順をまとめました。

1. 最初にインストールするもの
まずは、開発に必要なツールを自分のPCに入れてください。

Visual Studio Code (VS Code): コードを書くためのエディタ。

Git: コードの変更を記録・共有するツール。

Docker Desktop: 全員の開発環境を「全く同じ」にするためのツール。

インストール後、必ずアプリを起動して、クジラのマークが「緑色（Running）」になるまで待ってください。

2. プロジェクトへの参加手順
① コードを自分のPCに持ってくる（クローン）
GitHubのリポジトリページで、緑色の「Code」ボタンを押し、URLをコピーします。

VS Codeを開き、Ctrl + Shift + P を押して「git clone」と入力し、コピーしたURLを貼り付けます。

保存するフォルダを選べば、自分のPCにプロジェクトがコピーされます。

② 設定ファイルを作る（重要！）
パスワードなどはセキュリティ上GitHubに上がっていません。各自で以下の作業が必要です。

backend フォルダにある .env.example をコピーして、同じ場所に .env という名前で貼り付ける。

frontend フォルダにある .env.example をコピーして、同じ場所に .env.local という名前で貼り付ける。

※中身のパスワード等ができたら担当などに聞き合いましょう！

3. アプリの動かし方
VS Codeのターミナル（一番上のメニューから「新しいターミナル」）を開いて、以下のコマンドを打つだけです。

Bash
docker compose up --build
起動が完了したら、ブラウザで以下にアクセス！

画面の確認: http://localhost:3000

データの確認: http://localhost:8000

4. フォルダ構成の説明
どこをいじればいいかの地図です。

📂 frontend/: 見た目（React / Next.js）を作る場所

src/app/page.tsx: 最初に表示されるページ。

📂 backend/: データ処理（Python / FastAPI）を作る場所

main.py: サーバーの動きを書く場所。

📜 docker-compose.yml: 全体を動かすための指示書（基本はいじらなくてOK）。

5. 開発のルール（Gitの使い方）
main ブランチを直接汚さないために、以下の流れで作業します。

↓AIとかに聞いたら多分わかりやすく教えてくれる！もしわからなかったら聞いてください！

ブランチを作る: 作業前に「カレンダーのボタン作成」などの名前で新しいブランチを作ります。(mainでは作業しないように！)

コードを書く: VS Codeで楽しく開発！

コミット & プッシュ: 変更を保存してGitHubに送ります。

プルリクエスト（PR）を送る: GitHub上で「この変更を合体していい？」とリクエストを送ります。

困ったときは？
エラーが出たら、ターミナルの文字をコピーしてリーダーに送ってください。一緒に解決しましょう！
